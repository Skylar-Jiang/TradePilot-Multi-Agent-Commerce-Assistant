# 任务边界与验收总说明

## 1. 总体边界

当前项目按 `docs/daily-docs/课件第一天(1).docx`、`docs/daily-docs/课件第二天.docx` 以及现有架构文档对齐，目标是完成“可追溯数据 + Hugging Face RAG + OpenAI-compatible LLM + Tools/Agents/Memory + FastAPI 演示”的阶段闭环。

当前阶段不承诺完成：

- 绕过登录、验证码、反爬或 App 抓包的数据采集。
- Docker 容器化、云端一键部署。
- LangSmith 全链路追踪。
- 稳定获取真实批发市场分钟级成交价、库存、到货排队、冷链车辆轨迹和非公开采购合同数据。

这些内容属于后续部署、观测和授权数据接入阶段。

## 2. 当前完成情况

已完成的主体能力：

- 环境与依赖：`requirements.txt`、`.env.example`、`setup.py` 已覆盖 FastAPI、Chroma、OpenAI-compatible 客户端、Hugging Face embedding 相关依赖。
- 数据目录：`data/raw/`、`data/processed/`、`data/reports/` 已具备。
- 数据源配置：`config/sources.yaml` 与 `data/config/data_sources.json` 已包含生鲜批发采购公开源和区域供应源手工 CSV 样例。
- 数据读取：`modules/data_loader.py` 支持 CSV、公开网页、RSS。
- 清洗过滤：`modules/filtering.py`、`modules/semantic_filter.py` 支持规则过滤和语义过滤入口。
- RAG：`modules/rag_chain.py` 使用 Chroma + Hugging Face `BAAI/bge-m3`，支持 `HF_ENDPOINT` 镜像和模型名隔离的 collection。
- Tools：`modules/tools.py` 提供 CSV 导入、手工记录导入和 RAG 证据检索。
- Agents：`modules/agent_core.py` 提供价格、新品、舆情 Agent 和统一 Orchestrator。
- Chains：`modules/rag_chain.py`、`modules/prompts.py`、`modules/llm_client.py` 串起“证据检索 -> Prompt -> LLM -> JSON 解析”。
- Memory：`modules/memory_store.py` 提供分析缓存、Agent Trace 和会话消息管理。
- API：`modules/api_server.py` 暴露数据源、采集、RAG、Agent、Skill、Memory、报告和调度接口。

Provider 统一采用 OpenAI-compatible 协议。老师课件中的 Anthropic/Claude 示例在本项目中按 OpenAI SDK/兼容网关实现，可接 DeepSeek、SiliconFlow 或 OpenAI。

## 3. RAG 数据说明

RAG 需要真实或准真实信息才有分析价值。当前项目已有手工 CSV 样例、公开新闻/RSS/网页入口，可以验证完整架构；后续如果更换分析商品，只需要替换或新增数据源，再调用 `/rag/rebuild` 重建知识库。

数据采集边界：

- 推荐：公开网页、RSS、授权 API、公开样例 CSV、人工整理数据。
- 不推荐：绕过登录、验证码、反爬、App 抓包、付费或非公开内容。
- 对实时价格和具体时间点变化，只能在有合法稳定数据源时实现。

## 4. 环境配置

安装依赖前先进入虚拟环境：

```powershell
.\venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

DeepSeek 官方 OpenAI-compatible 配置示例：

```text
OPENAI_API_KEY=<你的 DeepSeek Key>
OPENAI_BASE_URL=https://api.deepseek.com/v1
MODEL_FAST=deepseek-v4-flash
MODEL_ANALYSIS=deepseek-v4-pro
MODEL_REPORT=deepseek-v4-pro
MODEL_TEMPERATURE=0.0
```

如果本地使用 `DEEPSEEK_API_KEY`、`DEEPSEEK_BASE_URL` 命名，`modules/llm_client.py` 也会读取并按 OpenAI-compatible 协议调用。

`MODEL_FAST`、`MODEL_ANALYSIS`、`MODEL_REPORT` 是分层 Agent 配置，不应强行统一。可根据账号可用模型调整为 DeepSeek 或其他 OpenAI-compatible 模型。

Hugging Face RAG 配置：

```text
HF_ENDPOINT=https://hf-mirror.com
RAG_EMBEDDING_MODEL=BAAI/bge-m3
RAG_EMBEDDING_DEVICE=cpu
RAG_NORMALIZE_EMBEDDINGS=true
```

如果镜像不可用，可以将 `HF_ENDPOINT` 改为 `https://huggingface.co` 或预先下载模型缓存。

## 5. 验收接口

推荐验收顺序：

1. `GET /`：服务健康检查。
2. `GET /config`：确认 LLM 与 RAG embedding 配置。
3. `GET /sources`：确认数据源配置可读。
4. `POST /ingest/csv`：导入 `data/raw/hema_price_manual.csv`。
5. `POST /rag/rebuild`：重建 Hugging Face + Chroma 知识库。
6. `GET /rag/search`：检索带来源的证据片段。
7. `GET /agents`：确认价格、新品、舆情 Agent。
8. `POST /analyze`：运行单 Agent 或 `agent=all`。
9. `GET /traces`：查看 Agent 调用轨迹。
10. `GET /memory/conversations/{session_id}`：查看会话记忆。

第一阶段重点看数据源、导入、RAG、看板；第二天重点看 Chains、Agents、Tools、Memory。

## 6. 本地验证

推荐命令：

```powershell
.\venv\Scripts\python.exe -m py_compile modules\rag_chain.py modules\llm_client.py modules\memory_store.py modules\api_server.py
.\venv\Scripts\python.exe tests_local.py
```

`tests_local.py` 使用轻量测试 embedding 验证本地链路，避免每次测试都下载 `BAAI/bge-m3`。真实运行 `/rag/rebuild` 时仍会使用 Hugging Face embedding。

## 7. 剩余风险

- `BAAI/bge-m3` 首次运行依赖 Hugging Face 或镜像下载；网络不稳定时会失败或超时。
- 当前网页采集只处理公开可访问页面，反爬严格或非公开的行情/监管/物流来源需要授权 API 或人工数据。
- Docker、云端一键部署和 LangSmith 追踪尚未进入当前完成范围。
