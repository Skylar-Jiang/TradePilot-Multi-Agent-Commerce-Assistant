### 模块一：竞品监控三大维度分析

#### 1.1 价格战监控（促销/定价变动）

价格战监控的核心是系统性追踪竞争对手的价格及相关数据，为企业自身决策提供依据。

**监控要点：**

- **价格变动记录**：记录竞品的原价、折扣价、优惠券力度等维度信息
- **促销活动追踪**：竞品是否参加了闪购、大促分会场等活动
- **预警机制**：当竞品某SKU价格降幅超过一定阈值时触发预警
- **策略判断**：判断竞品降价是清仓行为还是主动发起价格战

**实操建议：** 预设价格底线阈值，低于该阈值的调价建议必须经过人工审核方可执行，避免盲目跟价陷入恶性价格战。商情系统需实现实时数据流处理，通过Kafka+Flink架构构建数据管道，对竞品价格调整、新品发布等事件进行秒级响应。

#### 1.2 新品发布监控（产品迭代/功能上线）

新品发布监控帮助企业快速分析竞品的产品特点、目标用户和营销策略，进而调整自身的产品研发或营销计划。

**监控要点：**

- **新品预告追踪**：通过产品关键词自动实时抓取竞品发布的新品预告信息
- **功能迭代监测**：重点关注迭代功能、运营活动等关键动作
- **用户反馈采集**：新品上市后的用户反馈实时抓取，反哺下一代产品规划

**监控策略：** 抓大放小——重点跟踪核心竞品（3个以内），聚焦关键动作（迭代功能、运营活动、口碑变化），忽略无关细节（如界面微调、文案小改）。目前AI工具可覆盖产品动态（新品发布、功能迭代、价格调整）、营销动态、口碑动态、商业动态、危机动态等五大类公开信息的自动采集与分析。

#### 1.3 负面舆情监控（投诉/媒体报道）

负面舆情往往不是突然爆发的，而是在某些平台、某些话题中逐步积累的。

**监控要点：**

- **敏感预警**：实时监测全网信息，自动预警并精准研判一切负面舆情
- **传播分析**：建立竞品相关的新闻媒体监测和品牌舆情监控体系，与自身比较找出优劣势
- **趋势判断**：关注负面声量是否持续增长、是否在多个平台扩散

**核心监测指标：**

| 维度         | 监测内容            | 优先级 |
| ---------- | --------------- | --- |
| 品牌声量       | 社媒/论坛/新闻提及量、互动率 | 高   |
| SOV（声音占有率） | 竞品在行业话题中的声量占比   | 高   |
| 舆论倾向       | 正面/中性/负面舆论占比对比  | 中   |
| 用户反馈       | 核心诉求、吐槽点、认可点挖掘  | 中   |

#### 2.3 Document Loaders 概述

在 LangChain v1.x 中，文档加载器（Document Loaders）的核心抽象位于 `langchain-core` 包中。

**Document 对象的结构**：

- `page_content` (str)：文档的纯文本内容
- `metadata` (dict)：文档的元数据（如文件路径、来源URL、发布时间等）

**通用加载方法**：

- `load()`：将所有文档一次性加载到内存中
- `lazy_load()`：逐个产出Document，处理超大型文件时推荐使用

#### 2.4 使用底层库直接实现新闻、RSS、论坛数据加载

由于 `langchain-community` 已被淘汰，官方推荐对非合作伙伴的通用工具采用**直接封装底层API**的方式。以下使用 `feedparser`、`requests` + `BeautifulSoup` 等成熟库自行实现加载器，既轻量又避免了对已淘汰包的依赖。

**RSS Feed 加载（使用 feedparser）** ：

```python
import feedparser
from datetime import datetime
from langchain_core.documents import Document

def load_rss_feeds(feed_urls, continue_on_failure=True):
    """从RSS订阅源加载文章，返回Document列表"""
    documents = []

    for url in feed_urls:
        try:
            feed = feedparser.parse(url)
            for entry in feed.entries:
                # 提取内容
                content = entry.get('summary', '') or entry.get('description', '')
                if not content and 'content' in entry:
                    content = entry.content[0].get('value', '')

                # 提取发布时间
                pub_date = None
                if hasattr(entry, 'published_parsed') and entry.published_parsed:
                    pub_date = datetime(*entry.published_parsed[:6])

                doc = Document(
                    page_content=content,
                    metadata={
                        'source': url,
                        'source_type': 'rss',
                        'title': entry.get('title', ''),
                        'link': entry.get('link', ''),
                        'publish_date': pub_date.isoformat() if pub_date else '',
                    }
                )
                documents.append(doc)
        except Exception as e:
            if not continue_on_failure:
                raise
            print(f"加载RSS失败 {url}: {e}")

    return documents

# 使用示例
rss_urls = [
    "https://news.ycombinator.com/rss",
    "https://feeds.bbci.co.uk/news/rss.xml",
]
docs = load_rss_feeds(rss_urls)
print(f"加载了 {len(docs)} 篇文章")
```

**新闻页面加载（使用 requests + BeautifulSoup）** ：

```python
import requests
from bs4 import BeautifulSoup
from langchain_core.documents import Document

def load_news_urls(urls, continue_on_failure=True):
    """从新闻URL加载文章内容"""
    documents = []

    for url in urls:
        try:
            response = requests.get(url, timeout=10, headers={
                'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36'
            })
            response.encoding = 'utf-8'
            soup = BeautifulSoup(response.text, 'html.parser')

            # 提取标题和正文（需根据具体网站调整选择器）
            title = soup.find('h1')
            title_text = title.get_text(strip=True) if title else ''

            # 提取正文段落
            paragraphs = soup.find_all('p')
            content = '\n'.join([p.get_text(strip=True) for p in paragraphs])

            doc = Document(
                page_content=content,
                metadata={
                    'source': url,
                    'source_type': 'news',
                    'title': title_text,
                    'url': url,
                }
            )
            documents.append(doc)
        except Exception as e:
            if not continue_on_failure:
                raise
            print(f"加载新闻失败 {url}: {e}")

    return documents
```

```python
            else:

            doc = Document(
                page_content=content,
                metadata={
                    'source': url,
                    'source_type': 'forum',
                    'url': url,
                }
            )
            documents.append(doc)
        except Exception as e:
            if not continue_on_failure:
                raise
```

### 模块三：LangChain Text Splitters——竞品舆情文本的分块策略

#### 3.1 为什么需要文本分割

在LLM时代，模型上下文窗口有限，长文档无法一次性输入。文本分割的本质是在“信息完整性”与“模型处理能力”之间寻找平衡。

LangChain官方已将文本分割器独立为 `langchain-text-splitters` 包，即使不使用LangChain框架本身，也可以使用该包快速分割数据。

#### 3.2 TextSplitter核心参数

`TextSplitter` 是 `langchain-text-splitters` 包中的核心接口：

| 参数                | 类型        | 默认值  | 说明        |
| ----------------- | --------- | ---- | --------- |
| `chunk_size`      | int       | 4000 | 每个块的最大字符数 |
| `chunk_overlap`   | int       | 200  | 块之间的重叠字符数 |
| `length_function` | Callable  | len  | 测量块长度的函数  |
| `separators`      | List[str] | 见下文  | 分割符列表     |

#### 3.3 安装与导入

```bash

```


#### 3.4 RecursiveCharacterTextSplitter——通用首选

**核心设计理念**：优先使用大粒度分隔符（如段落`\n\n`），如果切出来的块仍超过`chunk_size`，退到下一级分隔符（如换行`\n`），继续退到更小粒度（空格、标点）。

**默认分隔符层级**：

```
["\n\n", "\n", "。", "，", " ", ""]
```

**基础用法**：

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

# 基础配置
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,          # 每块最大500字符
    chunk_overlap=100,       # 块间重叠100字符
    length_function=len,
    separators=["\n\n", "\n", "。", "，", " ", ""]
)

# 切割文档（传入Document列表）
chunks = text_splitter.split_documents(documents)

# 切割纯文本
text = "这是一段很长的竞品舆情分析文本..."
chunks = text_splitter.split_text(text)

print(f"共切割为 {len(chunks)} 个块")
for i, chunk in enumerate(chunks[:3]):
    print(f"Chunk {i}: {len(chunk)} 字符")
```

#### 4.2 完整的多源数据采集脚本（v1.x 架构）

```python
"""
竞品舆情数据采集脚本
功能：从新闻、RSS、论坛等多源采集竞品相关舆情数据
基于 LangChain v1.x 架构，不依赖已淘汰的 langchain-community
"""

import requests
import feedparser
from bs4 import BeautifulSoup
from datetime import datetime
from langchain_core.documents import Document
from langchain_text_splitters import RecursiveCharacterTextSplitter
import pandas as pd


class CompetitorMonitor:
    """竞品舆情监控采集器（基于 LangChain v1.x）"""

    def __init__(self):
        self.documents = []
        self.session = requests.Session()
        self.session.headers.update({
            'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36'
        })

    def fetch_news(self, urls, continue_on_failure=True):
        """采集新闻数据"""
        print(f"正在采集新闻数据，共 {len(urls)} 个URL...")
        count = 0
        for url in urls:
            try:
                response = self.session.get(url, timeout=10)
                response.encoding = 'utf-8'
                soup = BeautifulSoup(response.text, 'html.parser')

                title_tag = soup.find('h1')
                title = title_tag.get_text(strip=True) if title_tag else ''

                paragraphs = soup.find_all('p')
                content = '\n'.join([p.get_text(strip=True) for p in paragraphs])

                doc = Document(
                    page_content=content,
                    metadata={
                        'source': url,
                        'source_type': 'news',
                        'title': title,
                        'url': url,
                        'fetch_time': datetime.now().isoformat(),
                    }
                )
                self.documents.append(doc)
                count += 1
            except Exception as e:
                if not continue_on_failure:
                    raise
                print(f"  警告: 加载新闻失败 {url}: {e}")
        print(f"  采集到 {count} 条新闻")
        return count

    def fetch_rss(self, feeds, continue_on_failure=True):
        """采集RSS数据"""
        print(f"正在采集RSS数据，共 {len(feeds)} 个Feed...")
        count = 0
        for url in feeds:
            try:
                feed = feedparser.parse(url)
                for entry in feed.entries:
                    content = entry.get('summary', '') or entry.get('description', '')
                    if not content and 'content' in entry:
                        content = entry.content[0].get('value', '')

                    pub_date = None
                    if hasattr(entry, 'published_parsed') and entry.published_parsed:
                        pub_date = datetime(*entry.published_parsed[:6])

                    doc = Document(
                        page_content=content,
                        metadata={
                            'source': url,
                            'source_type': 'rss',
                            'title': entry.get('title', ''),
                            'link': entry.get('link', ''),
                            'publish_date': pub_date.isoformat() if pub_date else '',
                        }
                    )
                    self.documents.append(doc)
                    count += 1
            except Exception as e:
                if not continue_on_failure:
                    raise
                print(f"  警告: 加载RSS失败 {url}: {e}")
        print(f"  采集到 {count} 条RSS文章")
        return count

    def fetch_forum(self, urls, content_class=None, continue_on_failure=True):
        """采集论坛数据"""
        print(f"正在采集论坛数据，共 {len(urls)} 个URL...")
        count = 0
        for url in urls:
            try:
                response = self.session.get(url, timeout=10)
                response.encoding = 'utf-8'
                soup = BeautifulSoup(response.text, 'html.parser')

                if content_class:
                    elements = soup.find_all(class_=content_class)
                    content = '\n'.join([el.get_text(strip=True) for el in elements])
                else:
                    paragraphs = soup.find_all('p')
                    content = '\n'.join([p.get_text(strip=True) for p in paragraphs])

                doc = Document(
                    page_content=content,
                    metadata={
                        'source': url,
                        'source_type': 'forum',
                        'url': url,
                        'fetch_time': datetime.now().isoformat(),
                    }
                )
                self.documents.append(doc)
                count += 1
            except Exception as e:
                if not continue_on_failure:
                    raise
                print(f"  警告: 加载论坛失败 {url}: {e}")
        print(f"  采集到 {count} 条论坛帖子")
        return count

    def get_all(self):
        """获取所有已采集文档"""
        return self.documents

    def get_summary(self):
        """获取采集摘要"""
        summary = {
            'total': len(self.documents),
            'by_source': {}
        }
        for doc in self.documents:
            st = doc.metadata.get('source_type', 'unknown')
            summary['by_source'][st] = summary['by_source'].get(st, 0) + 1
        return summary


# 使用示例

    # 配置数据源（请替换为真实URL）

    summary = monitor.get_summary()
```

```python
import jieba
from collections import Counter

# 中文停用词（示例）
STOPWORDS = {'的', '了', '在', '是', '我', '有', '和', '就', '不', '人', '都', '一', '一个', 
             '上', '也', '很', '到', '说', '要', '去', '你', '会', '着', '没有', '看', '好', 
             '自己', '这', '那', '它', '他', '她', '我们', '你们', '它们', '什么', '怎么', 
             '为什么', '如何', '可以', '进行', '已经', '被', '把', '给'}

def extract_keywords(text, top_n=5):
    """提取关键词"""
    if not text or len(text) < 10:
        return []
    words = jieba.lcut(text)
    words = [w for w in words if w not in STOPWORDS and len(w) > 1]
    counter = Counter(words)
    return [word for word, _ in counter.most_common(top_n)]

df['keywords_extracted'] = df['content'].apply(lambda x: extract_keywords(x, 5))
```

```python
def auto_label(text):
    """
    基于关键词匹配的自动标注
    返回: (一级分类代码, 一级分类名称, 二级分类, 置信度)
    """
    if not isinstance(text, str):
        return ('other', '其他', '其他', 0.0)

    text_lower = text.lower()

    # 计算各维度的匹配得分
    scores = {}
    for category, info in LABELS.items():
        score = 0
        for keyword in info['keywords']:
            if keyword in text_lower:
                score += 1
        scores[category] = score

    # 选择得分最高的类别
    best_category = max(scores.keys(), key=lambda x: scores[x])
    best_score = scores[best_category]

    if best_score == 0:
        return ('other', '其他', '其他', 0.0)

    # 置信度：匹配关键词数量 / 该类别关键词总数
    confidence = best_score / len(LABELS[best_category]['keywords'])
    confidence = min(confidence, 1.0)

    return (best_category, LABELS[best_category]['name'], '其他', confidence)

# 对数据集进行标注
df['label_code'], df['label_name'], df['sub_label'], df['confidence'] = zip(
    *df['content'].apply(auto_label)

# 查看标注分布
```

3. **通用工具自行封装**：对非合作伙伴的通用工具（如 RSS 解析、网页抓取），直接使用底层库封装
4. **隔离 LangChain 代码**：将 LangChain 特定代码放在内部接口之后
