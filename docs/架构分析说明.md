# 架构分析说明

本项目原有 Skill、Agent、RAG、FastAPI、报告生成架构保持不变，研究对象统一为生鲜批发采购场景的区域供应源竞品动态追踪与智能对标分析。

架构链路：

1. 数据层：政府/行业价格页、批发市场行情、监管公告、天气物流信息、行业资讯、手工 CSV。
2. 清洗层：基础去噪、字段标准化、维度推断、文本分块。
3. RAG 层：使用 Chroma 存储分块后的生鲜批发行情和风险证据，保留来源 URL 与发布时间。
4. Agent 层：价格、供应动态、质量风险、趋势四类分析能力。
5. Skill 层：将 Agent 能力封装为可通过 Swagger 调用的业务 Skill。
6. 报告层：生成生鲜批发采购区域供应源竞品对标报告。
7. 服务层：FastAPI + Swagger 提供全功能测试入口。

这种设计说明研究对象可以替换，但通用工程能力可复用。当前不大规模重构字段名，`competitor` 字段继续作为“区域供应源竞品”的兼容字段使用。
