# 架构分析说明

本项目原有 Skill、Agent、RAG、FastAPI、报告生成架构保持不变，仅将研究对象切换为生鲜电商竞品动态追踪与价格舆情分析。

架构链路：

1. 数据层：政府/行业价格页、开放平台或授权 API、行业媒体、手工 CSV。
2. 清洗层：基础去噪、字段标准化、维度推断、文本分块。
3. RAG 层：使用 Chroma 存储分块后的生鲜情报，保留来源 URL 与发布时间。
4. Agent 层：价格、新品/节令、舆情、趋势四类分析能力。
5. Skill 层：将 Agent 能力封装为可通过 Swagger 调用的业务 Skill。
6. 报告层：生成生鲜电商竞品分析报告。
7. 服务层：FastAPI + Swagger 提供全功能测试入口。

这种设计说明研究对象可以替换，但通用工程能力可复用。
